## How to read

- A respect rate that climbs steeply ≤5pt → ≤20pt → zones are **near** pivots.
- A flat curve (e.g. 30 / 32 / 35%) → zones are **roughly in the area**, not aligned.
- A flat-low curve (e.g. <20% across all bands) → that zone type may be uncorrelated with pivots in this window.
- Compare across symbols to identify symbol-specific calibration issues.
- Compare across (tf, pattern_type) to identify which layer of the zone stack is the weak link.

## Caveats

- Pivot definition: fractal-5 (2 bars each side). Fractal-3 reported as count only for sensitivity.
- Historical zone state reconstructed from `valid_from` / `broken_at_date` / `valid_to`. Assumes these fields are not retroactively edited.
- `broken_at_date` is date-precision in schema — a zone broken intraday is treated as inactive from that date onwards. Pivots before the actual break on that date are conservatively excluded from that zone's respect count. Magnitude of bias: small (one-day boundary cases only).
- Respect = proximity. Does NOT test directional rejection (e.g. low pivot bouncing UP off a BULL_OB). Phase 2 candidate if proximity-based respect is high but trading edge is still low.
- Window straddles TD-029 era boundary. Era-aware TZ canonicalization applied.
- Zones with NULL `valid_from` excluded. Counts: NIFTY=0, SENSEX=0.
